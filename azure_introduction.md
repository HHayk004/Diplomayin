# Azure-ի և տվյալների մշակումին վերաբերող հիմնական հասկացությունները

## Ներածություն Azure-ի և ամպային հաշվարկային ծառայությունների վերաբերյալ

### Ամպային հաշվարկում (Cloud Computing)

Ամպային հաշվարկումը ինֆորմացիոն տեխնոլոգիաների մատուցման մոդել է, որտեղ հաշվարկային ռեսուրսները՝ սերվերներ, պահեստավորում (storage), տվյալների բազաներ, վերլուծական գործիքներ, ցանցային ռեսուրսներ և հավելվածներ, մատուցվում են ինտերնետի միջոցով՝ ըստ պահանջի։

### Ամպային մոդելի առավելությունները՝

- **Մասշտաբելիություն (Scalability)** – հնարավոր է արագ մեծացնել կամ նվազեցնել ռեսուրսները։
- **Ցածր նախնական ծախսեր** – ընկերությունները չեն գնում սեփական սերվերներ։
- **Բարձր հասանելիություն** – ծառայությունները աշխատում են աշխարհով մեկ բաշխված տվյալների կենտրոններում։
- **Անվտանգություն** – ամպային պլատֆորմներն ապահովում են ինքնաշխատ անվտանգություն և մոնիթորինգ։

### Microsoft Azure-ը

Microsoft Azure-ը Microsoft ընկերության զարգացրած համաշխարհային չափի ամպային հարթակ է։ Այն առաջարկում է հարյուրավոր ծառայություններ՝

- Համակարգչային հզորություն (Virtual Machines, Functions)
- Տվյալների պահեստավորում (Blob Storage, Data Lake)
- Տվյալների բազաներ (SQL Database, Cosmos DB)
- Մեքենայական ուսուցում (Azure ML)
- Տվյալների պրոցեսավորում և ինտեգրում (Data Factory, Synapse Analytics)
- Ցանցային կառավարման ծառայություններ և բազմաթիվ այլ գործիքներ

Azure-ը թույլ է տալիս ստեղծել, տեղակայել և կառավարել լուծումներ՝ առանց սերվերային ենթակառուցվածքի ֆիզիկական պահպանման։

## Ի՞նչ են ETL/ELT-ը, դրանց տարբերությունները և կիրառությունները

### ETL (Extract – Transform – Load)

ETL-ը տվյալների ինտեգրման դասական գործընթաց է, որտեղ.

- **Extract (Հեռացում)** – տվյալները վերցվում են տարբեր աղբյուրներից՝ տվյալների բազաներ, API-ներ, ֆայլեր։
- **Transform (Վերափոխում)** – տվյալները մաքրվում են, ձևափոխվում և բիզնես կանոններով մշակվում են միջանկյալ համակարգում։
- **Load (Բեռնում)** – արդեն վերափոխված տվյալները բեռնվում են նպատակային պահեստ՝ հաճախ՝ Data Warehouse։

**Օգտագործվում է, երբ՝**

- անհրաժեշտ է ծանր ու բարդ վերափոխումներ,
- նպատակային համակարգը չունի մեծ հաշվարկային հզորություն,
- տվյալները պետք է նախ մշակվեն, հետո պահվեն։

### ELT (Extract – Load – Transform)

ELT-ում գործընթացի հերթականությունը փոխված է․

- **Extract** – տվյալների հեռացում աղբյուրից
- **Load** – անմիջապես բեռնվում են նպատակային տվյալների պահեստ (օր.՝ Azure Synapse կամ Snowflake)
- **Transform** – վերափոխումն իրականացվում է հենց տվյալների պահեստի ներսում՝ օգտագործելով նրա հաշվարկային հզորությունը

**Օգտագործվում է, երբ՝**

- նպատակային պահեստն ունի ուժեղ հաշվարկային կարողություններ,
- անհրաժեշտ է աշխատել մեծ ծավալների հետ,
- տվյալների վերափոխումները կարելի է կատարել SQL-ով կամ Spark-ով անմիջապես պահեստում։

### ETL և ELT տարբերությունները

| Բնութագիր | ETL | ELT |
|----------|-----|-----|
| Վերափոխման տեղը | Միջանկյալ համակարգ | Տվյալների պահեստում |
| Առավել հարմար է | Փոքր/միջին համակարգերի համար | Մեծ տվյալների համակարգերի համար |
| Հաշվարկային ծանրաբեռնվածություն | ETL սերվերի վրա | Տվյալների պահեստի վրա |
| Արագություն | Ավելի դանդաղ մեծ տվյալների համար | Ավելի արագ մեծ տվյալների դեպքում |

## OLTP/OLAP

### OLTP (Online Transaction Processing)

OLTP համակարգերը նախատեսված են օպերացիոն, առօրյա, արագ և փոքր տրանզակցիաների համար՝ օրինակ՝ վճարումներ, գրանցումներ, պատվերների կատարում։

**Հատկություններ**

- Բազմաթիվ փոքր հարցումներ  
- Բարձր արագություն  
- Տվյալների խիստ ծավալային վերահսկում  
- Սովորաբար՝ նորմալիզացված տվյալների բազաներ  

**Օրինակներ**

- Բանկային համակարգեր  
- Առցանց խանութներ  
- CRM համակարգեր  

### OLAP (Online Analytical Processing)

OLAP համակարգերը նախատեսված են վերլուծության, հաշվետվությունների և մեծ ծավալի տվյալների վրա երկար հարցումների համար։

**Հատկություններ**

- Բարդ վերլուծական հարցումներ
- Ավելի փոքր հաճախականությամբ տրանզակցիաներ
- Տվյալների դենորմալիզացված կառուցվածք
- Օգտագործվում է BI, Dashboard-ներ, վերլուծական հաշվետվություններ ստեղծելու համար

**Օրինակներ**

- Azure Synapse Analytics
- Data Warehouse համակարգեր

## Տվյալային խողովակաշարեր (Data Pipelines)

Data Pipeline-ը տվյալների հոսքերի ավտոմատացված հավաքածու է, որը փոխանցում, մշակում և փոխակերպում է տվյալները աղբյուրներից դեպի նպատակային համակարգեր։

### Տվյալային խողովակաշարի հիմնական քայլերը

- Աղբյուրներից տվյալների հավաքագրում  
- Մաքրում և ձևափոխում (ETL/ELT)  
- Տվյալների տեղափոխում միջանկյալ կամ վերջնական պահեստներ  
- Մոնիթորինգ, սխալների կառավարում  
- Վերջնական օգտագործում՝ վերլուծության, մեքենայական ուսուցման և հաշվետվությունների համար  

Data Pipeline-ները կարևոր են, քանի որ ապահովում են տվյալների հուսալի, կրկնության դիմացկուն և ավտոմատացված շրջանառություն։

## Azure Data Factory — ներածություն և գործիքներ

### Azure Data Factory (ADF) ի՞նչ է

Azure Data Factory-ն Azure-ի ամպային տվյալների ինտեգրման ծառայություն է, որը թույլ է տալիս ստեղծել, կառավարել և ավտոմատացնել տվյալների տեղափոխման և փոխակերպման գործընթացները։

ADF-ն հիմնականում օգտագործվում է՝

- Տվյալների ETL/ELT գործընթացների համար  
- Տվյալային խողովակաշարեր կառուցելու համար  
- Տարբեր աղբյուրներից տվյալներ միացնելու համար  
- Տվյալների շարժի ամբողջական կառավարման համար (orchestration)  

### Azure Data Factory-ի հիմնական գործիքները

#### Pipelines
ADF-ի հիմնական կոնտեյներներն են, որտեղ հավաքվում են գործողություններ (activities), որոնք կատարում են տվյալների տեղափոխում, վերափոխում և այլ գործընթացներ։

#### Activities
Գործողություններ, որոնք կարող են լինել՝

- Copy Activity — տվյալների պատճենում տարբեր աղբյուրներից  
- Data Flow — տվյալների գրաֆիկական փոխակերպումներ  
- Stored Procedure Activity — SQL սքրիպտերի գործարկում  
- Web/API calls — API-ներից տվյալների ստացում  

#### Linked Services
Կապեր Azure-ի կամ արտաքին տվյալների աղբյուրների հետ (SQL Server, Blob Storage, Oracle և այլն)։

#### Datasets
Տվյալների կառուցվածքային նկարագրություն, որը ներկայացնում է թե ինչ տվյալ է ընթանում pipeline-ի մեջ։

#### Integration Runtime
Հաշվարկային ենթակառուցվածք, որը կատարում է տվյալների տեղափոխման և փոխակերպման աշխատանքը։  
Տեսակները՝

- Azure IR  
- Self-hosted IR  
- Azure SSIS IR

## Եզրակացություն

Azure-ը ամպային հարթակ է, որը առաջարկում է ամենաարդյունավետ լուծումներից մեկը տվյալների հետ աշխատելու համար։  
ETL և ELT գործընթացները ապահովում են տվյալների ինտեգրում, իսկ OLTP/OLAP համակարգերը՝ դրանց արդյունավետ պահպանումը և վերլուծումը։  
Azure Data Factory-ն հանդիսանում է այս ամբողջ էկոհամակարգի կարևոր գործիքը՝ թույլ տալով կառավարվել տվյալային հոսքերն ու ավտոմատացնել բարդ տվյալային գործընթացները։
